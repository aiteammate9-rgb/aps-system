# Reverse-Engineering myhora.com Life-Graph (เลข 12 ตัว)

สรุปผลการถอดพฤติกรรม (path B) ของ https://myhora.com/horoscope/life-graph.aspx
อ้างอิงข้อมูลที่เก็บจริงเมื่อ 2026-06-26 (~50 requests, มี delay 800ms)

## 1. วิธีดึงข้อมูล (ไม่ต้อง OCR)
- เรียกด้วย **GET**: `life-graph.aspx?d=&m=&y=&hh=&mm=&jd=&jm=&jy=&p=1`
  (d/m/y = วันเกิด สุริยคติ, y เป็น **พ.ศ.**, jd/jm/jy = วันจร) — ไม่ต้อง POST/VIEWSTATE
- กราฟคือ `<img src="image-year.aspx?d0..d11=&hy=&y=&by=">`
  → **เลข 12 ตัวโผล่ใน query string ตรง ๆ** (`d0..d11`), `hy`=อายุปัจจุบัน
- ค่ารายปีอยู่ใน `<map>` 12 `<area>`: `prophecy.aspx?id=<ค่า>&age=<อายุ>&...`

## 2. แกนกราฟ
- value (id) = จำนวนเต็ม **1..12** (1=ต่ำสุด/แย่, 12=สูงสุด/ดี)
- พิกเซล: `value = (442 - yTopPixel) / 34` (เช็คตรงเป๊ะทุกจุด)

## 3. โครงสร้างหลัก (ยืนยันด้วยข้อมูล)
- **value(age) = base[age mod 12]** — เป็นคาบ 12 ปีเป๊ะ (เช็ค age vs age+12 ตรงกันทุกคู่ ช่วงอายุ 13–60)
- `base[0..11]` = เวกเตอร์ 12 ค่าของบุคคล (index = age mod 12; index 0 = อายุ 0,12,24,...)

## 4. สูตร base (ลดรูปแล้ว)
```
base[h] = ((Q[weekday][h] - 1 + S) mod 12) + 1
```
- **weekday** = วันในสัปดาห์ของวันเกิด (อา..เสาร์) — พิสูจน์: 1 & 15 ส.ค. 2538 (อังคารทั้งคู่) ได้ค่าเท่ากัน, d=8 = d=1
- **Q[weekday][h]** = ตาราง 7×12 (เก็บครบแล้ว ที่ ref เดือน 8 / พ.ศ.2538):
  - houses 1–7 ขึ้นกับ weekday (ฐานวัน 7 ช่อง)
  - houses 0,8,9,10,11 ไม่ขึ้นกับ weekday
- **S** = ตัวเลื่อนสเกลาร์ (เลื่อนทั้ง 12 ช่องเท่ากัน):
  - **เดือน: +1 ต่อเดือน** (พิสูจน์ m8→m9 ทุกช่อง +1)
  - **ข้ามสงกรานต์ (มี.ค.→เม.ย.): +2** (เปลี่ยนปีนักษัตรพร้อมเดือน) — boundary อยู่ระหว่าง 28 มี.ค. กับ 4 เม.ย. 2538
  - **ปี: ~+1/ปี แต่ไม่สม่ำเสมอ** (+0/+1/+2 สลับ) → มาจากการดริฟต์ของวันเถลิงศก/ปีจุลศักราช (ต้องใช้โมเดลสุริยยาตร์/หรคุณเพื่อให้เป๊ะ)

## 5. ตาราง Q (ref: เดือน 8, พ.ศ. 2538) — base[0..11] ต่อวันในสัปดาห์
```
อาทิตย์: 7,10, 1, 4, 7,10, 1, 4,11, 1, 3, 5
จันทร์ : 7,11, 2, 5, 8,11, 2,10,11, 1, 3, 5
อังคาร : 7,12, 3, 6, 9,12, 8,11,11, 1, 3, 5
พุธ    : 7, 1, 4, 7,10, 6, 9,12,11, 1, 3, 5
พฤหัส  : 7, 2, 5, 8, 4, 7,10, 1,11, 1, 3, 5
ศุกร์   : 7, 3, 6, 2, 5, 8,11, 2,11, 1, 3, 5
เสาร์   : 7, 4,12, 3, 6, 9,12, 3,11, 1, 3, 5
```
(หมายเหตุ: houses 0,8,9,10,11 = 7,11,1,3,5 เหมือนกันทุกวัน = ส่วน S ของ ref นี้)

## 6. ภพ 12 (ตามที่เว็บแสดง)
วาสนา(ตนุ), ทรัพย์(กดุมภะ), เพื่อน(สหัชชะ), ญาติ(พันธุ), บริวาร(ปุตตะ),
ศัตรู(อริ), คู่ครอง(ปัตนิ), โรคภัย(มรณะ), ความสุข(ศุภะ), การงาน(กัมมะ),
ลาภยศ(ลาภะ), วินาศ(พยายะ)

## 7. สูตร S แบบ offline (สุริยยาตร์) — implement แล้วใน @life-graph/calendar
ค่าคงที่ (port จาก pythaidate): `CS_JULIAN_DAY_OFFSET=1954167`, `292207/800`, `+373`
```
horakhun = JDN - 1954167
masaken  = floor((horakhun + floor((horakhun*11 + 650)/692)) / 30)   // เดือนจันทรคติสะสม
zodiacYear(y,m) = y - (m < 4 ? 1 : 0)                                  // นักษัตร เปลี่ยนสงกรานต์
adhikamasa(cs)  = (tithi>24 || tithi<6) จาก horakhun ของปี cs         // ปีเดือน 8 สองหน
S = masaken + zodiacYear - (อธิกมาสสะสม)
base[h] = ((Q[weekday][h] - 1 + (S_birth - S_ref)) mod 12) + 1,  ref = 1 ส.ค. 1995
value(age) = base[age mod 12]
```

### ความแม่นที่ validate แล้ว (เทียบ ground truth จาก myhora)
- weekday sweep (ส.ค.1995, 7 วัน): **7/7 เป๊ะ**
- month sweep (12 เดือน ปี 1995 ข้ามสงกรานต์): **เป๊ะทั้งหมด** (ภายในปีนักษัตรเดียวกัน)
- year anchors (1970–2025, ราย 5 ปี): **8/11 เป๊ะ**
